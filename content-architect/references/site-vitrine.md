# Reference: Site Vitrine / Showcase Sites

## 1. When This Applies

Showcase sites, portfolios, corporate brochure sites, landing pages, freelancer sites, small business sites. Projects where the primary goal is to inform, showcase work, and convert visitors into leads or contacts -- not e-commerce, not SaaS, not content platforms.

## 2. Specific Inputs to Collect

Beyond the base SKILL.md inputs, ask:

- **Brand positioning**: high-end, accessible, technical, playful?
- **Primary conversion goal**: contact form, quote request, store visit, phone call, download?
- **Content assets available**: photos, testimonials, case studies, team bios, certifications?
- **SEO targets**: local SEO? What do people search for to find this business?
- **Blog/news content plan**: does the business have a real commitment to update? Who writes? How often?

## 3. Landing Page Frameworks

Do NOT default to AIDA. Use modern, evidence-based frameworks:

- **Julian Shapiro / Demand Curve**: Purchase Rate = Desire - (Labor + Confusion). Structure: Hero (descriptive header, NOT slogans) -> Social proof -> CTA -> Features & Objections -> CTA -> Footer. Key insight: "People don't have short attention spans -- they have short *consideration* spans."
- **PAS** (Problem-Agitate-Solution): Start with the specific problem instead of generic "Attention." Name the pain, twist the knife, then present the solution.
- **StoryBrand SB7** (Donald Miller, updated 2024): Customer = hero, brand = guide. Use the "3-step plan" to reduce confusion. Dual CTA strategy (direct + transitional): "marry me or date me" -- e.g., "Request a Quote" + "See Our Work."

## 4. Patterns

- **Hero + value proposition**: Above the fold: who you are + what you do + CTA in 5 seconds. Mobile-first: 83% of landing page traffic is mobile (Unbounce 2024) -- each screen-full needs one focal point.
- **Social proof placement**: Testimonials near CTAs, not on a dedicated page nobody visits. Stars/logos above the fold if available.
- **Service listing decision tree**: If each service needs >3 paragraphs of explanation, use separate pages. If not, use anchor sections on one page.
- **Contact as journey endpoint**: Every page has a visible path to contact. Contact is not hidden in the footer or behind 3 clicks.
- **F-pattern is a symptom, not a goal**: If users scan in F, the visual hierarchy has failed. Aim for "layer-cake pattern" -- clear headings that can be scanned independently. Test with the blur test (wexperience.fr): blur the page and check if the hierarchy is still visible.

## 5. Common Screens

- **Home**: Hero + value prop + social proof + CTA. Job: "Convince a visitor in 5 seconds that this is what they need."
- **About**: NOT an ego page. Job: "Build trust by showing the humans and values behind the work." Include: team, values, story only if it serves trust. Exclude: mission statements, corporate jargon.
- **Services / Products**: Grid or list. Job: "Help the visitor find the specific service they need." If >5 services with depth, consider individual pages.
- **Single Service page** (if depth warrants): Job: "Explain this service and move the visitor toward contact." Structure: problem -> solution -> proof -> CTA.
- **Portfolio / Realisations**: Grid with categories/filters. Job: "Show proof of quality work." Each item links to a detail page with context (client, challenge, result).
- **Contact**: Form + phone + address + map. Job: "Make it easy to reach out." Always include a confirmation page after submission.
- **Legal** (mentions legales, privacy policy): Required by law. Not in main nav -- footer link only.
- **Blog/News**: ONLY if committed to updating. A stale blog (last post 18 months ago) is worse than no blog. If no content plan exists, do not add a blog page.

## 6. Worked Example

A freelance plumber who wants a site to show his work and receive quote requests.

**Personas**:
1. Homeowner with an urgent plumbing issue
2. Homeowner planning a bathroom renovation

**Journeys**:
- Urgent: Google "plombier [city]" -> Home -> Contact (or direct phone link). This journey must complete in <30 seconds.
- Renovation: Google "renovation salle de bain [city]" -> Home -> Realisations -> single project -> Contact/Devis.
- Trust-building: Home -> About -> Avis clients -> Contact.

**Screens** (7 total): Home, Realisations, Single project detail, Services, Contact/Devis, About, Legal.

**Nav** (4 items): Home | Realisations | Services | Contact.

**MVP**: Home, Realisations (even 3-4 photos), Contact/Devis. Everything else is v2.

## 7. Pitfalls

- **The About page trap**: Too much ego, too little relevance. Nobody reads the founder's life story. Show team photos, certifications, years of experience -- things that build trust, not things that feed vanity.
- **The Services dump**: Every service listed with equal weight, no prioritization. Visitors drown. Lead with the top 3 services that drive the most revenue, then link "all services" for completeness.
- **The blog graveyard**: "We'll update it" -- they won't. A blog with the last post from 2023 signals abandonment. Only add a blog page if there is a real content plan with assigned ownership and a publishing cadence.
- **Hidden contact**: Contact form buried behind About -> Team -> Contact. Every page needs a visible CTA or footer link to contact. The phone number belongs in the header on mobile.
- **Desktop-first thinking**: 83% of landing page traffic is mobile. Design for thumb zones, not mouse hovers. Test every layout on a real phone screen, not just a resized browser window.
