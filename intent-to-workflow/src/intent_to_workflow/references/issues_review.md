# Issues Review Gate

Review `issues.md` only. Do not create `workflow.md`, `tracker.md`, prompts, or next-phase artifacts during this gate.

Check `issues.md` against `prd.md`:

- each issue is a vertical slice with a verifiable outcome;
- dependencies are explicit and ordered;
- AFK/HITL routing is justified;
- acceptance criteria and validation are concrete;
- TDD notes identify the first behavior test;
- agent guidance is scoped enough for future handoff.

Report either approval or blocking findings. If corrections are needed, describe the exact change and wait for the human decision before editing.
